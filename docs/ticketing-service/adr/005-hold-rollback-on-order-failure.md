# ADR 005 — 주문 생성 실패 시 Redis 선점 즉시 롤백

**날짜**: 2026-06-18
**상태**: 확정

---

## 배경

좌석 선점(Redis) 성공 후 order-service Feign 호출이 실패하는 경우를 어떻게 처리할지 결정이 필요했다.

---

## 결정: 주문 생성 실패 시 Redis DEL로 좌석을 즉시 AVAILABLE로 복원

---

## 이유

선점 성공 후 Feign 호출이 실패했을 때 롤백하지 않으면, 주문 없이 좌석만 `HOLDING` 상태로 600초(TTL) 동안 묶여버린다. 실패를 감지한 즉시 Redis에서 선점 관련 키를 지우고 재고를 복원해야 다른 유저가 바로 그 좌석을 다시 시도할 수 있다.

## 참고

이 결정 시점 이후 hold/checkout 분리(2026-07-03)로 실제 롤백 로직의 위치는 `hold()`에서 `checkout()`으로 이동했다. [architecture.md §5](../architecture.md#5-동시성-제어-전략) 참고.
