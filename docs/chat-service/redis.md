# Chat Service Redis 사용

## Redis 사용

| 용도 | 키 예시 | 설명 |
| --- | --- | --- |
| 방 멤버 캐시 | `chat:room:{roomId}:members` | 브로드캐스트 대상 조회용 캐시(캐시 어사이드, TTL+지터) |
| 접속 상태(presence) | STOMP 세션 기준 | 접속 중이면 실시간 전달, 아니면 푸시 발행 |
| 슬로우 모드 | `chat:slow:{roomId}:{userId}` | 쿨다운 |
| 도배 억제 | `chat:last:{roomId}:{userId}` | 직전 메시지 해시 |

방 멤버 캐시는 조회 시 미스면 DB에서 적재하고, 멤버 변경 이벤트 시 갱신/무효화한다.

구현 세부:

- 멤버 캐시는 Redis Set이며 TTL은 기본 3600초 + 0~600초 지터(랜덤)를 더해 만료를 분산한다(캐시 스탬피드 방지).
- 접속 상태 조회는 `SMISMEMBER`로 여러 사용자를 한 번에 검사한다(N회 개별 조회 대신).
- 도배 감지는 직전 메시지 원문이 아니라 `content.hashCode()` 문자열을 저장·비교한다(저장 공간 절약, 이론상 해시 충돌 가능성 존재).
